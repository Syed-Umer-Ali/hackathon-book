# Quickstart: Book Chapter Content Organization

**Date**: 2025-11-29

This document provides a quick guide to understanding and contributing to the "Physical AI & Humanoid Robotics Textbook" content structure within Docusaurus.

## Content Structure Overview

The textbook's content is organized hierarchically within the `physical-ai-book/docs/` directory.

-   **Top-Level Documents**: Found directly in `physical-ai-book/docs/` (e.g., `intro.md`, `requirements.md`).
-   **Modules (Chapters)**: Reside in numbered subdirectories (e.g., `physical-ai-book/docs/01-module-1-ros2/`).
-   **Lessons (Sub-sections)**: Are Markdown files within module directories (e.g., `physical-ai-book/docs/01-module-1-ros2/1-lesson-title.md`).

## Adding New Lessons/Content

To add a new lesson or content page:

1.  **Locate the appropriate Module directory**: Navigate to `physical-ai-book/docs/<module-directory>/`.
2.  **Create a new Markdown file**: Name the file using a numeric prefix for ordering, followed by a descriptive slug (e.g., `X-your-lesson-title.md`).
3.  **Add Front Matter**: At the top of the new Markdown file, include Docusaurus front matter:
    ```markdown
    ---
    title: Your Lesson Title
    sidebar_position: X
    ---
    ```
    Replace `X` with the sequential position of the lesson within its module.
4.  **Add Content**: Write your lesson content in Markdown below the front matter.
5.  **Save the file**: Docusaurus's autogenerated sidebar will automatically detect the new page.

## Modifying Existing Content

1.  **Locate the Markdown file**: Find the `.md` file corresponding to the content you wish to modify.
2.  **Edit the content**: Update the Markdown text as needed.
3.  **Save the file**: Changes will be reflected when the Docusaurus site is rebuilt or reloaded in development mode.

## Updating Navigation Order

-   **Lessons within a Module**: Adjust the `sidebar_position` in the file's front matter or change the numeric prefix of the filename.
-   **Modules (Chapters)**: Change the numeric prefix of the module's directory name (e.g., rename `01-module-1-ros2` to `05-new-module`).

## Local Development

To view changes locally:

1.  Navigate to `physical-ai-book/`.
2.  Run `npm start`.
3.  Open your browser to `http://localhost:3000`.

This quickstart will help maintainers and contributors effectively manage the textbook's content.
